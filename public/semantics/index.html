  • HTML tags have NO PRESENTATIONAL VALUE! Don't even think of them like that. They are only there to lend meaning and context to the content. This page is constructed using only unordered lists. No other markup was employed to make this page. This page illustrates how CSS should provide the presentation – regardless of what tags are used to construct it.
  • Don't use this particular method for constructing your own pages, however. Using nothing but <ul>s and <li>s to make your page is silly and meaningless, unless the content is nothing but lists. The moral of this story is that you should use HTML tags ONLY for their semantic value – for how the tag defines the content's place in the context of the page. Presentation is the realm of CSS.
